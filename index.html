Overview

LT-VQA is a controlled multi-session dataset for evaluating long-term object-centric scene understanding. Unlike single-session VQA benchmarks, LT-VQA targets persistent cross-session object identity, event-level state transition annotation, and session-indexed temporal reasoning — enabling questions such as "Where has the green chair been across all sessions?"

Event Taxonomy

Ground-truth annotations cover 5 atomic event types per object per session, plus 4 compound query types requiring multi-session reasoning.

Atomic Events
APPEAR
Object first observed in session St
DISAPPEAR
Object no longer detected after St
MOVE
Centroid displacement beyond threshold
RE-APPEAR
Object returns after prior disappearance
NONE
No state change detected in session St
Compound Query Types (Multi-Session Reasoning)
Trajectory
Sequence of transitions across all sessions
Volatility
Frequency and magnitude of changes
Temporal Localization
Identifying when a change occurred
Counterfactual
Comparing states at two specific sessions

Qualitative Results

Example QA Pairs

Delta Memory — Event History
Q: "What happened to the robot dog at Session 6?" Delta Memory
A: At Session 6, the robot dog moved from (−0.81, 0.44, 1.77) to (−0.96, 2.52, −2.27).
Q: "Where has the green chair been across all sessions?" Delta Memory
A: S1: Appear (2.8, 0.9, 0.2) → S5: Move (−1.0, 1.0, −1.2) → S8: Disappear → S9: Re-Appear (2.6, 0.9, 0.1).
Live Memory — Current State
Q: "Where did the trashbin end up after all sessions?" Live Memory
A: At the end (S10), the trashbin was at (−0.64, 0.52, 1.50).
Q: "Where is the green chair now?" Live Memory
A: At (2.96, 0.88, 0.25) as of Session 10.
Meta Memory — Long-Term Statistics
Q: "Which object moved most frequently?" Meta Memory
A: The brown basket (4 moves).
Q: "How many times did the robot dog move?" Meta Memory
A: The robot dog moved 2 times over 10 sessions. Classified as semi-static.
